# TKMM CLI Extension API — v1

**Status:** Current  
**Extension ID:** `TKVSC-Team.tkvsc-tkmm-cli`  
**API version:** `1` (`api.apiVersion`)

This extension bundles **Tkmm.CLI** and exposes it to other VS Code extensions so each addon does not need its own copy of the binary.

## Overview

| What this extension provides | What it does *not* provide |
|------------------------------|----------------------------|
| Bundled `Tkmm.CLI` per platform | TKVSC project tree, RomFS, or format handlers |
| `TkmmCliApi` from `activate()` | In-editor UI inside the `.tkproj` webview |
| Shared output channel (**TKMM CLI**) | A published npm types package (copy `consumer/` files for now) |

**Typical consumer flow:**

1. Declare `extensionDependencies` on `TKVSC-Team.tkvsc-tkmm-cli`
2. Call `getTkmmCliApi()` (or `getExtension(...).activate()`)
3. Resolve a project path (via this API or [TKVSC `resolveProjectRoot`](https://github.com/TKVSC-Team/totk-vscode/blob/main/docs/api/v1.md))
4. Call `packageMod()` or `runCli()`

## Quick start

### 1. `package.json`

```json
{
  "extensionDependencies": [
    "TKVSC-Team.totk-vscode",
    "TKVSC-Team.tkvsc-tkmm-cli"
  ]
}
```

`TKVSC-Team.totk-vscode` is only required if you use `resolveProjectContext()` with TKVSC tree items. Path-based APIs work without it.

### 2. Copy consumer helpers

Until a shared npm package exists, copy into your addon:

| File | Purpose |
|------|---------|
| [`consumer/tkmmCliTypes.ts`](../../consumer/tkmmCliTypes.ts) | Types + constants |
| [`consumer/getTkmmCliApi.ts`](../../consumer/getTkmmCliApi.ts) | Safe API acquisition |

Keep these in sync with this document when the API changes.

### 3. Use in `activate()`

```typescript
import * as vscode from 'vscode';
import { getTkmmCliApi } from './getTkmmCliApi';

export async function activate(context: vscode.ExtensionContext) {
  const tkmm = await getTkmmCliApi();
  if (!tkmm) {
    return;
  }

  if (!tkmm.isCliAvailable()) {
    void vscode.window.showWarningMessage(
      'TKMM CLI is not available. Install the TKVSC TKMM CLI extension or set tkvscTkmmCli.cliPath.',
    );
    return;
  }

  context.subscriptions.push(
    vscode.commands.registerCommand('my-addon.package', async () => {
      const project = tkmm.resolveProjectFromPath('/path/to/mod');
      if (!project) return;

      const result = await tkmm.packageMod({
        projectPath: project.projectRoot,
        outputPath: `${project.projectRoot}/${tkmm.defaultTkclFileName(project.projectRoot)}`,
        showOutputChannel: true,
      });

      if (result?.success) {
        void vscode.window.showInformationMessage(`Packaged: ${result.outputPath}`);
      }
    }),
  );
}
```

## Obtaining the API

### Recommended: `getTkmmCliApi()`

```typescript
import { getTkmmCliApi } from './getTkmmCliApi';

const api = await getTkmmCliApi();
```

Validates extension presence, activation, and `api.apiVersion === 1`. Shows error toasts unless `{ silent: true }`.

### Direct activation

```typescript
const ext = vscode.extensions.getExtension('TKVSC-Team.tkvsc-tkmm-cli');
const api = await ext?.activate();

if (!api || api.apiVersion !== 1) {
  // unsupported or missing
}
```

**Activation order:** List `TKVSC-Team.tkvsc-tkmm-cli` in `extensionDependencies` so it activates before your addon.

## Versioning

### `api.apiVersion`

```typescript
readonly apiVersion: 1
```

Increments only on **breaking** changes to `TkmmCliApi`. Always guard:

```typescript
if (api.apiVersion !== 1) {
  return;
}
```

`getTkmmCliApi()` does this automatically.

### `api.extensionId`

Always `'TKVSC-Team.tkvsc-tkmm-cli'`.

## `TkmmCliApi` interface

```typescript
interface TkmmCliApi {
  readonly apiVersion: 1;
  readonly extensionId: string;

  getCliPath(): string | undefined;
  isCliAvailable(): boolean;

  resolveProjectContext(item?: unknown): Promise<TkmmProjectContext | undefined>;
  resolveProjectFromPath(projectPath: string): TkmmProjectContext | undefined;
  defaultTkclFileName(projectRoot: string): string;

  runCli(options: TkmmCliRunOptions): Promise<TkmmCliRunResult>;
  packageMod(options: TkmmPackageOptions): Promise<TkmmPackageResult | undefined>;

  showOutputChannel(): void;
}
```

## CLI resolution

### `getCliPath()`

Returns the absolute path to `Tkmm.CLI`, or `undefined` if none is configured for the current platform.

Resolution order:

1. `tkvscTkmmCli.cliPath` setting (if non-empty)
2. Bundled binary under this extension's `bin/` folder:
   - Windows: `bin/Tkmm.CLI.exe`
   - macOS / Linux: `bin/Tkmm.CLI`

### `isCliAvailable()`

`true` when `getCliPath()` is set **and** the file exists on disk.

```typescript
const tkmm = await getTkmmCliApi();
if (!tkmm?.isCliAvailable()) {
  void vscode.window.showErrorMessage('TKMM CLI binary not found.');
  return;
}
```

### `getCliPath()` only (spawn yourself)

If you need custom process handling:

```typescript
import { spawn } from 'child_process';

const cliPath = tkmm.getCliPath();
if (!cliPath) return;

const child = spawn(cliPath, ['mods', 'install', modPath], { cwd: projectRoot });
```

Prefer `runCli()` for consistent path resolution and error handling.

## Project resolution

### `TkmmProjectContext`

```typescript
interface TkmmProjectContext {
  projectRoot: string;   // folder containing .tkproj
  tkprojPath: string;    // absolute path to .tkproj file
}
```

### `resolveProjectFromPath(projectPath)`

Synchronous. Accepts:

- A **project root** directory (must contain `.tkproj`)
- A **`.tkproj` file** path

Returns `undefined` if no valid `.tkproj` exists.

```typescript
const ctx = tkmm.resolveProjectFromPath('D:/mods/MyMod');
const ctx2 = tkmm.resolveProjectFromPath('D:/mods/MyMod/.tkproj');
// both return { projectRoot, tkprojPath } when valid
```

### `resolveProjectContext(item?)`

Async. Resolves from, in order:

1. **TKVSC tree item** passed to a command handler (`resourceUri` + `resolveProjectRoot` via TKVSC API)
2. A tree item whose `resourceUri` is a `.tkproj` file
3. The **active text editor** when it is a `.tkproj` document

Returns `undefined` when no project can be determined.

```typescript
// Context menu command on TKVSC Projects tree
vscode.commands.registerCommand('my-addon.package', async (treeItem) => {
  const project = await tkmm.resolveProjectContext(treeItem);
  if (!project) return;
  // ...
});

// No argument — uses active .tkproj editor
const project = await tkmm.resolveProjectContext();
```

Requires TKVSC for tree-item resolution. Active-editor fallback works without TKVSC.

### `defaultTkclFileName(projectRoot)`

Returns a sanitized filename from `.tkproj` → `Mod.Name`, e.g. `My Cool Mod.tkcl`, or `mod.tkcl` on parse failure.

## Running the CLI

### `runCli(options)`

```typescript
interface TkmmCliRunOptions {
  args: string[];                        // arguments after the executable
  cwd?: string;                        // working directory
  onOutput?: (chunk: string) => void;  // stdout + stderr
}

interface TkmmCliRunResult {
  exitCode: number;
}
```

Spawns `Tkmm.CLI` with the given args. Does not show UI unless you handle `onOutput` or open the output channel yourself.

**Known subcommands** (subject to TKMM CLI changes):

| Invocation | Description |
|------------|-------------|
| `['mods', 'package', projectRoot, output]` | Package project folder to `.tkcl` |
| `['mods', 'install', ...]` | Install a mod |
| `['mods', 'package-optimizer', ...]` | Package optimizer |

```typescript
const { exitCode } = await tkmm.runCli({
  args: ['mods', 'package', project.projectRoot, outputPath],
  cwd: project.projectRoot,
  onOutput: (text) => myChannel.append(text),
});

if (exitCode !== 0) {
  void vscode.window.showErrorMessage(`TKMM CLI exited with code ${exitCode}`);
}
```

Throws if the CLI binary is missing.

## Packaging mods

### `packageMod(options)`

High-level wrapper around `mods package`.

```typescript
interface TkmmPackageOptions {
  projectPath: string;           // project root or .tkproj path
  outputPath?: string;           // .tkcl destination
  promptForOutput?: boolean;     // show save dialog if outputPath omitted
  saveTkprojFirst?: boolean;     // default true — save dirty .tkproj editor
  showOutputChannel?: boolean;     // default false — log to TKMM CLI channel
  onOutput?: (chunk: string) => void;
}

interface TkmmPackageResult {
  success: boolean;              // exitCode === 0
  exitCode: number;
  projectRoot: string;
  tkprojPath: string;
  outputPath: string;
}
```

**Returns `undefined`** when the user cancels the save dialog (`promptForOutput: true`).

**Throws** when:

- `projectPath` has no `.tkproj`
- CLI binary is missing
- `outputPath` is omitted and `promptForOutput` is not `true`

### Examples

**Silent programmatic package:**

```typescript
const result = await tkmm.packageMod({
  projectPath: root,
  outputPath: `${root}/out.tkcl`,
  saveTkprojFirst: true,
  onOutput: (line) => console.log(line),
});
```

**With save dialog (like the built-in command):**

```typescript
const result = await tkmm.packageMod({
  projectPath: root,
  promptForOutput: true,
  showOutputChannel: true,
});
```

**Equivalent CLI invocation:**

```text
Tkmm.CLI mods package <projectRoot> <output.tkcl>
```

The first argument is the **project folder** (containing `.tkproj`, `romfs/`, etc.), not the `.tkproj` file itself.

Working directory is set to the project root.

### `showOutputChannel()`

Opens the **TKMM CLI** output channel. Useful after failed `runCli()` / `packageMod()` calls.

## Integration with TKVSC

Combine both APIs for project-tree commands:

```typescript
import { getTkvscApi } from './getTkvscApi';
import { getTkmmCliApi } from './getTkmmCliApi';

const tkvsc = await getTkvscApi();
const tkmm = await getTkmmCliApi();
if (!tkvsc || !tkmm) return;

vscode.commands.registerCommand('my-addon.packageFromTree', async (treeItem) => {
  const root = tkvsc.resolveProjectRoot(treeItem);
  if (!root) return;

  const adapter = tkvsc.detectProjectAdapter(root);
  if (adapter.id !== 'tkmm') {
    void vscode.window.showWarningMessage('Not a TKMM project.');
    return;
  }

  await tkmm.packageMod({
    projectPath: root,
    promptForOutput: true,
    showOutputChannel: true,
  });
});
```

You can also use `tkmm.resolveProjectContext(treeItem)` instead of `tkvsc.resolveProjectRoot()` when you only need the `.tkproj` paths.

## Settings

Consumers inherit user configuration from this extension:

| Setting | Type | Default | Description |
|---------|------|---------|-------------|
| `tkvscTkmmCli.cliPath` | `string` | `""` | Override path to `Tkmm.CLI`. Empty uses bundled binary. |

Users configure this in VS Code settings; consumer extensions do not need their own CLI path setting unless they want a separate override.

## Error handling

| Situation | Behavior |
|-----------|----------|
| Extension not installed | `getTkmmCliApi()` → `undefined`, error toast |
| API version mismatch | `getTkmmCliApi()` → `undefined`, error toast |
| CLI missing on disk | `isCliAvailable()` → `false`; `runCli` / `packageMod` throw |
| Invalid project path | `resolveProjectFromPath` → `undefined`; `packageMod` throws |
| User cancels save dialog | `packageMod` → `undefined` |
| Non-zero CLI exit | `runCli` → `{ exitCode }`; `packageMod` → `{ success: false, exitCode }` |

**Recommended pattern for user-facing commands:**

```typescript
try {
  const result = await tkmm.packageMod({ ... });
  if (!result) return; // cancelled
  if (!result.success) {
    tkmm.showOutputChannel();
    void vscode.window.showErrorMessage(`Packaging failed (exit ${result.exitCode}).`);
  }
} catch (error) {
  const msg = error instanceof Error ? error.message : String(error);
  void vscode.window.showErrorMessage(msg);
}
```

## Built-in command reference

This extension also contributes `tkvsc-tkmm-cli.packageAsTkcl` (UI only). Consumers can invoke the same behavior via `packageMod({ promptForOutput: true, showOutputChannel: true })` or register their own commands.

| UI location | `when` clause |
|-------------|---------------|
| `.tkproj` editor title bar | `activeCustomEditorId == 'totk-editor.tkprojEditor'` |
| Projects sidebar | `view == totk-editor.archives` + project / `.tkproj` context values |
